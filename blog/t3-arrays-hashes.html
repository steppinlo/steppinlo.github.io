Arrays vs Hashes


1/10/2015

When it comes to arrays vs hashes, they appear to do similar things but they actually are completely different. In short, arrays are a collection of data, whereas hashes have pairs/assignments. Do note that when you do use a hash or an array they have different symbols, hashes are with {}, whereas arrays use [].

To go further in depth, arrays have wide number of values, which can literally be anything (e.g. integers, strings, etc), each object can be referenced from its specific location. It should be noted that the first object will always start from 0, i.e. array[0] would be the first object in the array. Also, a nice thing to note is if there is an unknown number of attributes within the array, the last object can be called upon with [-1]. Example of an array is below:

my_array = ["This", "is", "an", "array"]

Each element can be called with their numbers, so if I had a command that said, puts my_array[0], it would have "This" printed on the screen.

Hashes, on the other hand, will have assignments already linked to each object. So, for example, if you have a hash set up as below

class="indent"> state_capitol = { "New York" => "Albany"

"California" => "Sacramento"

"Illinois" => "Springfield" }

As you can see above, I have the states and the capitols set up. So if this was a collection of all 50 states, I could simply have in my ruby code: "puts state_capitol['New York']" and it would print out Albany.

There are benefits to using each. Arrays are better put to use when you have an ordered list of things or a collection, such as grades over a semester for a particular student. Hashes are better when there is an assignment to each category. For example you might want to use hashes if you are classifying an object to a category (e.g. sport team names to the specific sport).
